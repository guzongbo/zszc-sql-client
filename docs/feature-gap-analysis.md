# 功能差距分析 (Feature Gap Analysis)

本报告旨在将 `zszc-sql-client` 与市面上流行的数据库客户端（如 Navicat, DataGrip, DBeaver, TablePlus）进行功能对比，找出当前产品的核心功能缺失，并为未来的开发路线提供优先级建议。

## 1. 现有核心功能盘点

根据代码分析，本项目当前已经具备一个坚实的基础，核心功能包括：

- **数据库连接与管理：**
  - 支持 MySQL 数据库连接。
  - 提供连接配置的保存、分组和管理功能。
  - 通过树状图导航数据源、数据库和数据表。
- **数据操作与编辑：**
  - 在网格中查看和编辑表数据。
  - 支持手动事务模式下的数据变更。
  - 提供 SQL 预览并能应用数据变更（增、删、改）。
- **结构设计与维护：**
  - 查看和编辑表结构。
  - 支持创建新表。
  - 可查看表的 DDL（Data Definition Language）。
  - 提供 SQL 预览并能应用表结构变更。
- **SQL 控制台：**
  - 内置带自动补全功能的 SQL 编辑器。
  - 支持执行 SQL 查询和变更，并能查看结果。
- **核心特色 - 数据库对比：**
  - **数据对比：** 先进的多阶段数据对比引擎，支持全量或选择性表对比，能展示差异详情并生成同步 SQL。支持任务进度跟踪和取消。
  - **结构对比：** 可对比两个数据库间的表结构差异（新增、修改、删除），展示 DDL 差异并生成同步 SQL。
- **导入/导出：**
  - 支持从 Navicat 导入连接配置。
  - 支持将对比结果的同步 SQL 导出到文件。
- **历史记录：**
  - 本地保存和展示历史对比记录。

## 2. 功能差距与开发优先级

尽管核心对比功能强大，但作为一款通用数据库客户端，`zszc-sql-client` 在一些基础和进阶功能上仍有缺失。以下是按优先级排列的功能差距清单：

### 优先级：高 (High Priority) - 核心竞争力与基础功能

这些是用户在选择数据库客户端时最基本、最普遍的期望，也是扩展用户群体的关键。

| 缺失功能 | 功能描述 | 建议理由 |
| :--- | :--- | :--- |
| **多数据库支持 (PostgreSQL, SQLite)** | 在现有 MySQL 的基础上，增加对 PostgreSQL 和 SQLite 这两个主流开源数据库的支持。| 这是扩展用户群体的最有效方式。许多开发者同时使用多种数据库，仅支持 MySQL 会使用场景受限。PostgreSQL 和 SQLite 是最常见的组合。|
| **SSH 隧道 / SSL 连接** | 提供通过 SSH 隧道连接到远程数据库服务器的能力，并支持 SSL 加密连接。| 这是连接生产环境或云数据库的**必备安全功能**。缺少此功能会使大量专业用户无法在工作中使用本工具。 |
| **通用数据导出 (CSV, JSON)** | 允许用户将数据表或查询结果直接导出为通用的数据格式，如 CSV 和 JSON。| 这是最基本的数据交换需求。目前应用只能导出用于同步的 SQL，而无法导出用于分析或交付的数据本身。|

### 优先级：中 (Medium Priority) - 提升效率与专业性

这些功能虽然不是必备，但极大地提升了开发和分析效率，是从“可用”到“好用”的标志。

| 缺失功能 | 功能描述 | 建议理由 |
| :--- | :--- | :--- |
| **ER 图 (Entity-Relationship Diagram) 查看器** | 将数据库中的表和它们的外键关系以图形化的 ER 图形式展示出来。| 对于理解复杂数据库的结构至关重要。一个只读的 ER 图查看器就能极大地提升用户体验，帮助用户快速理清数据关系。|
| **可视化查询构建器 (Visual Query Builder)** | 提供一个拖拽式的图形化界面，让用户可以通过选择表、字段和条件来生成复杂的 SQL 查询。| 显著降低了编写复杂 SQL 的门槛，对 SQL 不熟练的用户非常友好，同时也能提高 SQL 专家的工作效率。|
| **数据导入向导** | 提供一个引导式界面，帮助用户将 CSV 或 JSON 文件的数据导入到数据库的表中。| 与数据导出相辅相成，完善了数据进出的闭环。可以简化数据迁移和初始化的过程。|

### 优先级：低 (Low Priority) - 锦上添花与特定场景

这些功能更偏向于数据库管理（DBA）或特定工作流的深度优化。

| 缺失功能 | 功能描述 | 建议理由 |
| :--- | :--- | :--- |
| **代码片段/模板** | 允许用户保存常用的 SQL 语句作为代码片段，方便随时调用。| 提升重度 SQL 用户的编写效率。|
| **用户与权限管理** | 提供图形化界面来管理数据库的用户、角色和权限授予（GRANT/REVOKE）。| 这是 DBA 的常用功能，对于普通开发者来说需求频率较低。|
| **备份与恢复向导** | 提供一个图形化界面来调用 `mysqldump` 等原生工具，执行数据库的备份和恢复操作。| 虽然方便，但许多团队已经有自己的备份策略和脚本，工具的替代性较强。|

## 总结

`zszc-sql-client` 的数据库对比功能是其强大的差异化优势。为了使其成为一款功能更全面、更具竞争力的通用数据库客户端，建议的开发路线图应首先聚焦于补齐**高优先级**中的核心功能，即：**扩展数据库支持、完善安全连接、提供通用数据导出**。

完成这些基础功能后，再投入资源开发**中优先级**的效率工具（如 ER 图和可视化查询构建器），将能显著提升产品的专业性和用户体验。
